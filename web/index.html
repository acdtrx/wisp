Wisp KVM & containers

Run VMs and containers from one clean panel

Wisp is a deployable web app for managing QEMU/KVM virtual machines via libvirt and OCI workloads via containerd—live host stats, storage, networking, consoles, backups, and more.

What you get

Functionality

◇

Virtual machines

Full lifecycle control over libvirt domains: CPU, memory, disks, networks, snapshots, cloud-init, USB passthrough, and SPICE/VNC console access.

▢

Containers

OCI containers alongside VMs in one workload list—containerd-backed, with images, mounts, and networking aligned with the same patterns as the rest of the app.

◎

Host & live data

CPU, memory, disk and network I/O pushed over Server-Sent Events—no polling the UI for stats, jobs, or workload state.

◈

Operations

Image library, SMB-aware backups, app and host configuration, and a focused layout built for daily ops on a single machine or homelab server.

How it was made

Wisp is vibe coded. The UI, API, and automation were built end to end in Cursor—pair programming with AI for structure, refactors, and polish—while keeping a single coherent design language: light surfaces, clear typography, and calm blue accents—the same look you see in the product.
